Product Overview

ArbitrageIQ is a B2C SaaS product that sells on-demand PDF market analysis reports for Short-Term Rental (STR) investors. Users search a city or select a market bundle, pay once via Stripe, and receive professionally formatted PDF report(s) instantly by email — no account required, no subscription. The product targets rental arbitrage investors who need market intelligence (nightly rates, occupancy, seasonality, regulatory risk) before committing to a lease.

Frontend Tech Stack

Next.js 16
App Router
React framework with file-based routing, server components, and API routes built in. Eliminates the need for a separate backend server.
One codebase handles UI, API routes, and webhook endpoints
TypeScript
Strict
Typed superset of JavaScript. Catches bugs at compile time rather than runtime — critical when handling payment data and PDF generation.
Zero runtime type errors in payment + PDF flow
Tailwind CSS v4
Utility-first
Utility-first CSS framework. Styles live in component markup — no separate stylesheet files, no naming conflicts, instant dark mode support.
Fast UI iteration without context switching
React 19
UI Layer
Component-based UI library. Server Components reduce JavaScript sent to the browser. Client Components used only where interactivity is required.
Minimal client JS — faster load times
@react-pdf/renderer
Server only
Renders React component trees into PDF documents. Used server-side only (incompatible with the DOM). Produces professional, branded PDF reports.
Programmatic PDF — no external PDF service needed
Geist Font
Local
Vercel's design system font, loaded from the local npm package. Avoids Google Fonts network calls that caused infinite dev server hangs on Windows.
Reliable local font loading, no external dependency

Service Stack

Stripe
Payments
Handles all payment processing via hosted Checkout sessions. No card data touches our servers. Webhook delivers checkout.session.completed events to trigger PDF generation and email delivery.
PCI compliant out of the box, zero payment infrastructure to maintain
Resend
Email
Transactional email API purpose-built for developers. Sends PDF attachments reliably with simple base64 encoding. Domain verification ensures strong deliverability.
Developer-friendly API, high deliverability, generous free tier
Vercel
Hosting
Serverless hosting platform with native Next.js support. Auto-deploys from GitHub on every push to main. Zero-config SSL, edge network, and built-in environment variable management.
Git push = production deploy, no DevOps required
Cloudflare
DNS + CDN
Manages DNS for arbitrage-iq.com. Provides DDoS protection, fast global DNS propagation, and free SSL certificates. Acts as a CDN layer in front of Vercel.
Fastest DNS propagation, free DDoS protection, free tier
GitHub
Source Control
Hosts the source repository. Vercel watches the main branch and deploys automatically on every push. Also provides the remote source for Claude Code scheduled agents to clone.
Triggers Vercel deploys + enables remote agent access
Claude Code (Anthropic)
AI Agents
Scheduled remote agents run in Anthropic's cloud. The Weekly Market Research agent clones the repo, web-searches current STR data for all 15 markets, and commits a structured update report every Sunday at 12am ET.
Keeps static market data fresh without manual research effort

Data Architecture

markets.json
Static · Server-side
Single source of truth for all market data. Contains 15 markets with nightly rates, occupancy, seasonality, competition level, regulatory risk, and Go/Caution/Avoid verdict. Case-insensitive city+state matching. To add a market: add one JSON entry — no code changes required.
Zero database infrastructure, instant reads, version-controlled data
waitlist.json
File-backed · Runtime
Captures email + city for users requesting unsupported markets. Created at runtime. Appropriate for MVP/single-instance deployments. Not suitable for multi-region serverless at scale — migrate to a database (PlanetScale, Supabase) when needed.
Zero-infrastructure demand capture for MVP validation

Agent Architecture

A parent-child agent model keeps each agent focused and context-lean. Child agents run independently on schedules, write structured outputs to agents/outputs/{name}/report.md, and never modify production data directly. A parent agent (future) will synthesize all child outputs into a single weekly action report.
Agent Type Schedule Output Status
Weekly Market Research Child Sunday 12am ET agents/outputs/market-research/report.md Live
Parent Synthesis Agent Parent Sunday 6am ET agents/PARENT_REPORT.md Planned

Replication Insights — Building the Next Product

  • 01
    Static JSON > database for MVP data A JSON file checked into the repo is faster to build, free to host, and version-controlled. Only migrate to a real database when you have concurrent writes or multi-instance deployments. For read-heavy products, static data is almost always the right MVP choice.
  • 02
    Stripe webhook is the backbone of digital goods delivery The pattern — Checkout session → webhook → fulfill — is battle-tested. Always return HTTP 200 from webhooks even on errors to prevent Stripe retries. Log errors for manual investigation. Never trust that payment metadata will survive the round trip without explicit storage.
  • 03
    Commit before you deploy Vercel deploys whatever is on GitHub — not your local machine. Forgetting to push is the #1 cause of "my changes aren't showing up" and "no framework detected" errors. Always verify the GitHub repo reflects your local state before connecting to Vercel.
  • 04
    Use live Stripe keys from day one in production Test keys produce test transactions that never settle. The moment you point a real domain at your app, switch to live keys. Register a production webhook endpoint against the real URL immediately — without it, no orders fulfill.
  • 05
    Anchor pricing drives bundles The Pro Bundle ($97, save $188) exists primarily to make the Analyst Pack ($47) feel like a bargain. Classic anchoring. Place the "Most Popular" badge on the option you want to sell most. Showing per-report pricing ($15.67 vs $19) makes the value concrete.
  • 06
    AI agents replace manual research workflows A weekly Claude agent that researches and reports is cheaper and more consistent than manual data gathering. The parent-child architecture — each agent writes one file, parent synthesizes — keeps context lean and agents composable. Design for this from day one.
  • 07
    CLAUDE.md is the handoff document Every architecture decision, known gap, naming convention, and flow should be in CLAUDE.md. A new Claude Code session with a thorough CLAUDE.md picks up in minutes rather than hours. Treat it as a living document — update it after every meaningful session.
  • 08
    Vercel + Cloudflare + Resend is the zero-ops stack This combination covers hosting, CDN, DNS, email, and SSL with zero server management. Total fixed cost under $5/month. Scales to thousands of users before any infrastructure changes are needed. Use it as the default stack for every new B2C digital product.
  • 09
    Test the full flow with your own card before launch Type checking and unit tests verify code. They do not verify that Stripe webhooks fire, that PDFs generate correctly, or that emails land in inboxes. Always do a real end-to-end purchase to your own email before going public. It takes 2 minutes and catches real issues.
  • 10
    Conversion placement matters more than copy Moving the pricing section above the feature grid produced an immediate layout improvement. Users should see the offer before they see the justification. Hero → Price → Social Proof → Features is consistently the highest-converting order for digital products.
